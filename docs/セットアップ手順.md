# セットアップ手順（ユーザー作業）

このとおりに上から進めれば、本番で動く状態になる。所要目安 20〜30分。
コマンドは基本コピペでよい。**プロジェクトフォルダのパス**は自分の環境に読み替えること。

> 旧Slack配信は廃止済み。手順から Slack Webhook は消えた（Vercelの `SLACK_WEBHOOK_URL` は削除してよい）。

---

## 0. 前提

- GitHub / Vercel / Neon（すべて設定済みなら手順3から）
- Googleアカウント（ログインに使うもの。会社用・個人用どちらでも可）

---

接続文字列2種を `.env` に貼り → `cd ~/work/myfolda/AIニュース && npx prisma db push`

## 2. NEXTAUTH_SECRET を生成する ✅ ローカルは済み

ローカルの `.env` には生成・設定済み。**Vercel用に同じ値を使う**ので、`.env` の `NEXTAUTH_SECRET` の値を控える。
（新しく作る場合は `openssl rand -base64 32`）

---

## 3. Google CloudでOAuthクライアントを作る【いちばん重要】

アプリへのログイン（Googleサインイン）に必要。フェーズ2のカレンダー・Gmail連携でも同じものを使う。

1. https://console.cloud.google.com を開き、上部のプロジェクト選択 → **新しいプロジェクト**（名前例: `ai-hisho`）
2. 左メニュー **APIとサービス → OAuth同意画面**
   - User Type: **外部（External）** → 作成
   - アプリ名（例: `AI秘書`）・ユーザーサポートメール・デベロッパー連絡先を入力 → 保存して次へ
   - スコープ: 何も追加せず次へ（ログインだけなら不要。連携スコープはフェーズ2で追加）
   - **テストユーザー**: ログインに使う自分のメールを追加（会社用・個人用など全部）→ 保存
   - ※ 公開ステータスは「テスト」のままでよい（個人利用のため）
3. 左メニュー **APIとサービス → 認証情報 → 認証情報を作成 → OAuthクライアントID**
   - アプリケーションの種類: **ウェブアプリケーション**
   - 承認済みのリダイレクトURI に**2つ**追加:
     - `http://localhost:3000/api/auth/callback/google`
     - `https://あなたのアプリ.vercel.app/api/auth/callback/google`（VercelのURLに読み替え）
   - 作成 → 表示される **クライアントID** と **クライアントシークレット** を控える
4. ローカルの `.env` に貼る:
   - `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` = 手順3の値
   - `ALLOWED_EMAIL` = ログインを許可する自分のメール（複数ならカンマ区切り）
5. 動作確認: `npm run dev` → http://localhost:3000 → Googleでログインできれば成功

---

## 3b. Google連携（Gmail・カレンダー）を有効にする【フェーズ2】

同じGCPプロジェクトで3つの作業をする。

1. **APIを有効化**（左メニュー **APIとサービス → ライブラリ**）:
   - `Gmail API` を検索 → 有効にする
   - `Google Calendar API` を検索 → 有効にする
2. **リダイレクトURIを追加**（**認証情報 → 作成済みのOAuthクライアントを開く**）。
   「承認済みのリダイレクトURI」に**2つ追加**（ログイン用と合わせて合計4つになる）:
   - `http://localhost:3000/api/google/callback`
   - `https://あなたのアプリ.vercel.app/api/google/callback`
3. **暗号化キーを生成して `.env` に追加**（ターミナルで1行）:
   ```
   echo "TOKEN_ENCRYPTION_KEY=\"$(openssl rand -hex 32)\"" >> ~/work/myfolda/AIニュース/.env
   ```
4. 動作確認: `npm run dev` → 設定ページ → 表示名を入れて「Googleアカウントを連携」→
   Googleの同意画面（「確認されていません」警告は**続行**でOK）→ 設定に「接続中」で戻れば成功。
   会社用・個人用それぞれで実行する。家族共用カレンダーは個人アカウントの「カレンダーを選ぶ」でチェック。

> 連携アカウントは**GCPのテストユーザーに載っているメール**である必要がある（手順2で追加済みのはず）。
> テストモードのため連携は**7日で失効**する。切れたら設定の「再連携」を1タップすれば復活する。

## 4. Vercelの環境変数を更新する

**Settings → Environment Variables**（Production対象）:

| Key | 値 |
|---|---|
| `NEXTAUTH_SECRET` | 手順2の値（ローカルと同じでよい） |
| `NEXTAUTH_URL` | `https://あなたのアプリ.vercel.app` |
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` | 手順3の値 |
| `ALLOWED_EMAIL` | 許可する自分のメール |
| `TOKEN_ENCRYPTION_KEY` | 手順3bの値（ローカルの .env と同じ値にする） |
| （既存）`DATABASE_URL` `DIRECT_URL` `CRON_SECRET` `DEEPL_API_KEY` `ANTHROPIC_API_KEY` | 変更なし |
| ~~`SLACK_WEBHOOK_URL`~~ | **削除してよい** |

## うまくいかないとき

| 症状 | 確認する場所 |
|---|---|
| ログイン画面に「セットアップが必要です」 | 表示された環境変数名を .env / Vercel に設定したか |
| Googleログイン後に「許可されていません」 | `ALLOWED_EMAIL` にそのメールが入っているか（大文字小文字は無視される） |
| ログインが `redirect_uri_mismatch` で失敗 | GCPのリダイレクトURIがアプリURLと一致しているか（http/https・末尾まで正確に） |
| ダッシュボードが「DBに接続できません」 | `DATABASE_URL` の値、`prisma db push` を実行したか |
| GitHub Actionsが401 | `APP_URL`/`CRON_SECRET` のSecretsがVercel側と一致しているか |
