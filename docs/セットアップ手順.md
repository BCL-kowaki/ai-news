# セットアップ手順（ユーザー作業）

このとおりに上から進めれば、本番で動く状態になる。所要目安 20〜40分。
コマンドは基本コピペでよい。**プロジェクトフォルダのパス**は自分の環境に読み替えること。

配信先は **Slack**。

---

## 0. 前提

- GitHubアカウント
- Vercelアカウント（GitHub連携でログイン可）
- Neonアカウント（無料）
- Slackワークスペース（無料でOK。配信を受け取るチャンネルを1つ用意）

---

## 1. Neon（データベース）を用意する ✅ 済み

> ステップ1として完了済み。テーブル作成（`prisma db push`）まで確認済み。
> 再セットアップ時のみ以下を実施する。

1. https://neon.tech で New Project を作成（Region: Singapore 等）
2. 接続文字列を2つ控える:
   - **Pooled connection**（`-pooler` あり）→ `DATABASE_URL`
   - **Direct connection**（`-pooler` なし）→ `DIRECT_URL`
3. `.env` に貼り、`cd ~/work/myfolda/AIニュース && npx prisma db push`

---

## 2. Slack Incoming Webhook を発行する

1. https://api.slack.com/apps を開き、**Create New App → From scratch**
2. App名（例: `AIニュース`）と、配信先の**ワークスペース**を選んで Create
3. 左メニュー **Incoming Webhooks** を開き、**Activate Incoming Webhooks** を On
4. 下の **Add New Webhook to Workspace** をクリック
5. 配信を受け取る**チャンネル**を選んで Allow
6. 発行された **Webhook URL**（`https://hooks.slack.com/services/T.../B.../xxxx`）をコピー
   → これを教えてください（`.env` と Vercel に設定します）

> このURL自体が投稿権限そのもの。**秘密情報**として扱う（Gitには載せない）。

---

## 3. CRON_SECRET を生成する

ターミナルで:

```
openssl rand -hex 32
```

出てきた文字列を控える（VercelとGitHubに同じ値を入れる）。

---

## 4. GitHubにpushする

まだGitHubリポジトリが無ければ作成し、このフォルダをpushする。
`.env` は `.gitignore`済みなので**アップロードされない**（秘密情報は安全）。

```
cd ~/work/myfolda/AIニュース && git push
```

---

## 5. Vercelにデプロイ + 環境変数

1. https://vercel.com で **Add New → Project** → 上のGitHubリポジトリを import
2. Framework は Next.js が自動選択される。そのまま Deploy
3. デプロイ後、**Settings → Environment Variables** に次の4つを登録（Production対象）:

| Key | 値 |
|---|---|
| `DATABASE_URL` | Neonの Pooled 接続文字列 |
| `DIRECT_URL` | Neonの Direct 接続文字列 |
| `SLACK_WEBHOOK_URL` | 手順2のWebhook URL |
| `CRON_SECRET` | 手順3で生成した文字列 |

4. 環境変数を足したら **Deployments → 最新 → Redeploy** で反映
5. 発行されたURL（例 `https://ai-news-xxxx.vercel.app`）を控える = **APP_URL**

---

## 6. GitHub Actions（定期実行）を有効化

1. GitHubリポジトリ → **Settings → Secrets and variables → Actions → New repository secret** で2つ登録:
   - `APP_URL` = 手順5のVercel URL（末尾スラッシュなし）
   - `CRON_SECRET` = 手順3と同じ値
2. リポジトリの **Actions** タブを開き、ワークフローを有効化（初回は「I understand my workflows, go ahead」）
3. 動作確認: Actions → 「AIニュース定期実行」→ **Run workflow** →
   - `fetch` を選んで実行 → 成功すれば `/`（ダッシュボード）で「未送信の記事」が増える
   - 次に `broadcast` を実行 → 選んだSlackチャンネルにニュースが届けば完成 🎉

以降は自動で、毎時収集・1日3回（8/13/19時）配信される。

---

## うまくいかないとき

| 症状 | 確認する場所 |
|---|---|
| ダッシュボードが「DBに接続できません」 | `DATABASE_URL` の値、`prisma db push` を実行したか |
| Slackに届かない | `SLACK_WEBHOOK_URL` が正しいか、Vercelで Redeploy したか。Vercel Logsで `[配信]` を確認 |
| GitHub Actionsが401 | `APP_URL`/`CRON_SECRET` のSecretsがVercel側と一致しているか |
| 記事が0件のまま | `fetch` を先に実行したか（配信は収集済みの記事を流すため） |
