# 開発整理ドキュメント（開発ルール・ファイルリスク区分）

> 最終更新: 2026-07-15

## ファイルのリスク区分

変更前に必ずこの区分を確認する。**高リスク**は勝手に変えず、必ず承認を取る。

### 🔴 高リスク（触るときは必ず承認・慎重に）
| ファイル | 理由 |
|---|---|
| `src/app/api/cron/broadcast/route.ts` | Slackへ投稿する場所。投稿順・sent更新・件数制御が命 |
| `src/lib/auth.ts` | Cron認証。緩めると第三者にエンドポイントを叩かれる |
| `prisma/schema.prisma` | DBスキーマ。変更はマイグレーションを伴う |
| `.env` / Vercel・GitHubのSecrets | 秘密情報（Webhook URL・DB接続・CRON_SECRET）。漏洩・誤設定で停止や乗っ取り |

### 🟡 中リスク（影響範囲を確認して変更）
| ファイル | 理由 |
|---|---|
| `src/lib/config.ts` | 全体の定数。配信件数・分割・間隔に直結する |
| `src/lib/format.ts` | Slack整形・分割・エスケープ。壊すとメッセージ崩れ |
| `src/lib/slack.ts` | Slack投稿の本体 |
| `src/lib/datetime.ts` | JST判定。ずれると配信時刻の見出しや集計がずれる |
| `src/lib/source-sync.ts` | ソース同期。無効化ロジックを壊すと収集対象が変わる |
| `.github/workflows/cron.yml` | 実行スケジュール |

### 🟢 低リスク（自由に変更してよい）
| ファイル | 内容 |
|---|---|
| `src/lib/sources.ts` | RSSソース定義。増減は1行追加/削除でOK（DBは自動同期） |
| `src/app/page.tsx` | ダッシュボード表示 |
| `docs/` | ドキュメント |

## 開発ルール（このプロジェクト固有）

- **定数は必ず `src/lib/config.ts` に集約する。** ローカル定数を作らない
- **RSSソースの追加・削除は `src/lib/sources.ts` の編集だけ。** DB作業もマイグレーションも不要（実行時に自動同期・削除分は自動無効化）
- **秘密情報はコードに書かない。** 必ず環境変数
- **Slackに載せる外部由来のテキスト（記事タイトル）は必ずエスケープする**（`format.ts` の escapeSlack）
- 完了報告の前に `npx tsc --noEmit` と `npx next build` を通す
- 分岐ロジックには意図をコメントで残す（特に順序に意味がある箇所）
