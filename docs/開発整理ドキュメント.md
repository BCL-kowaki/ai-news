# 開発整理ドキュメント（開発ルール・ファイルリスク区分）

> 最終更新: 2026-07-17（秘書エージェント化 フェーズ1）

## ファイルのリスク区分

変更前に必ずこの区分を確認する。**高リスク**は勝手に変えず、必ず承認を取る。

### 🔴 高リスク（触るときは必ず承認・慎重に）
| ファイル | 理由 |
|---|---|
| `src/middleware.ts` | 全ページのログイン必須化。matcherを緩めると誰でも入れる／絞りすぎるとcronが止まる |
| `src/lib/nextauth.ts` | ログイン許可の判定（許可リスト・フェイルセーフ）。緩めると第三者がログインできる |
| `src/lib/auth.ts` | Cron認証。緩めると第三者にエンドポイントを叩かれる |
| `prisma/schema.prisma` | DBスキーマ。変更はマイグレーションを伴う |
| `.env` / Vercel・GitHubのSecrets | 秘密情報。漏洩・誤設定で停止や乗っ取り |
| `src/lib/crypto.ts` | refresh_token暗号化（AES-256-GCM）。壊すと全連携が復号不能になる |
| `src/lib/google/oauth.ts` / `src/app/api/google/**` | OAuthフロー・CSRF対策。refresh_tokenは実質パスワード |

### 🟡 中リスク（影響範囲を確認して変更）
| ファイル | 理由 |
|---|---|
| `src/lib/config.ts` | 全体の定数・ジャンル定義・優先度定義 |
| `src/app/(main)/*/actions.ts` / `src/app/actions.ts` | Server Actions。認証チェックと入力ガードを崩さない |
| `src/lib/datetime.ts` | JST判定。ずれると期限ラベル・dateKeyがずれる |
| `src/lib/source-sync.ts` | ソース同期。無効化ロジックを壊すと収集対象が変わる |
| `.github/workflows/cron.yml` | 実行スケジュール |
| `tailwind.config.ts` / `src/app/globals.css` | デザイントークンの単一定義元。全画面に影響 |

### 🟢 低リスク（自由に変更してよい）
| ファイル | 内容 |
|---|---|
| `src/lib/sources.ts` | RSSソース定義。増減は1行追加/削除でOK（DBは自動同期） |
| `src/app/(main)/**/page.tsx` | 画面表示 |
| `src/components/` | UI部品 |
| `docs/` | ドキュメント |

## 開発ルール（このプロジェクト固有）

- **定数・分類は必ず `src/lib/config.ts` に集約する。** ローカル定数を作らない
- **色・角丸などのトークンは `globals.css` のCSS変数が単一定義元。** Tailwind側は名前の割り当てのみ
- **デザインはエディトリアル調を維持する**（参考: yuji.uk/html/260713。紙色#F8F3E8・茶枠2px・ずらし影・レンガ赤#C85E47・極太見出し・パステル+茶枠チップ）
- CSS変数色に透明度修飾子（`/60`等）は使えない。透明度は変数に焼き込むかリテラル色で書く
- **RSSソースの追加・削除は `src/lib/sources.ts` の編集だけ**（実行時に自動同期・削除分は自動無効化）
- **秘密情報はコードに書かない。** 必ず環境変数。画面には「未設定のキー名」までしか出さない
- **Server Actionは必ず先頭でセッション確認**（assertLoggedIn。ミドルウェアがあっても省略しない）
- **認証系のフェイルセーフを守る**: 環境変数が未設定なら「動かない」側に倒す
- アイコンは絵文字ではなくlucide-react（SVG）を使う
- 完了報告の前に `npx tsc --noEmit` と `npx next build` を通す
- 分岐ロジックには意図をコメントで残す（特に順序に意味がある箇所）

```

```
