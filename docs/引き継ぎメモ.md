## ハマりどころ（実際に起きた/起きうる問題と対策）

### 1. Vercel Hobbyプランのcronは「1日1回」まで【重要】
- 毎時fetch・1日3回broadcastはVercel無料プランのcronでは**デプロイ時にエラー**（公式確認済み）。
- **対策**: 定期実行を **GitHub Actions**（無料・分単位可）にし、Vercelはエンドポイントとして叩かれる構成。
- Vercel Proに上げるなら `vercel.json` に `crons` を足してActions側を止める（仕様書に記載）。

### 2. VentureBeatのAIフィードが更新停止していた
- 最新記事が約56日前で、48時間の鮮度フィルタで全部落ちていた（バグではない）。
- **対策**: The Verge AI・MIT Tech Review に差し替え。`syncSources` に「定義から消えたソースを自動でactive=falseにする」処理を追加したので、`sources.ts` の編集だけでソースを完全に増減できる。

### 2b. Google News（q=AI）のノイズ・重複問題
- 実配信で判明。178件中111件がGoogle News由来で、日本語のPRリリース（PR TIMES等）・同一ニュースの重複（複数媒体が同じ記事を転載）・芸能ノイズが大量に混入していた。
- **対策**: 方針を「一次情報・技術中心」に決定し、Google Newsを除外。TechCrunch / The Verge / The Decoder / MIT Tech Review / OpenAI / Google AI Blog / Ars Technica / ITmedia の8ソースに再構成。除外後は71件の良質な記事になった。
- 日本語ソースはITmedia AI+が中心（編集記事でPRが少ない）。日本語を増やしたい場合は、PRの少ない編集系の日本語AIメディアを追加検討。

### 3. RSSの末尾スラッシュとリダイレクト
- URL末尾の `/` 有無で308リダイレクトになるフィードがある（rss-parserは基本追う）。取得0件が続くソースはURLとリダイレクトをまず疑う。

### 4. Neonは接続文字列が2種類
- 実行時は **Pooled**（`-pooler`付き）を `DATABASE_URL`、マイグレーションは **Direct** を `DIRECT_URL`。

### 5. Next.jsの型キャッシュが削除ファイルを参照する
- ルートを削除した後、`.next/types` に古い参照が残り `tsc` が失敗することがある。`rm -rf .next` してから再ビルドで解消。
