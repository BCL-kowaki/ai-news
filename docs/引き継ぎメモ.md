## ハマりどころ（実際に起きた/起きうる問題と対策）

### 6. Next.js 14.2.15にミドルウェア認証バイパスの重大CVE【今回対応済み】
- `x-middleware-subrequest` ヘッダでミドルウェアを素通りできる脆弱性（CVE-2025-29927）。ログイン必須化が無意味になるところだった。
- **対策**: 14.2.35へ更新済み。`npm audit` に残る警告はNext 15/16でしか直らないもの（DoS系中心）。将来Next 15へ上げる。

### 7. FormData の priority が Number(null)=0 になる罠【今回対応済み】
- ダッシュボードのクイック追加には優先度欄が無く、`Number(formData.get("priority"))` が `Number(null)=0`（低）になっていた。
- **対策**: `formData.get()` の null を先に判定してからデフォルト「中(1)」を適用。

### 8. Server Actionのフォームは送信後に入力が残る
- `<form action={serverAction}>` はNext 14では自動リセットされない。
- **対策**: `src/components/AutoResetForm.tsx`（アクション完了後に form.reset() する客側ラッパー）を全入力フォームで使用。

### 9. GCP同意画面「テストモード」のrefresh tokenは7日で失効（フェーズ2の前提）
- Gmail readonlyはrestrictedスコープで、個人利用は審査を通さずテストモード運用が現実解。ただしテストモードのトークンは7日で切れる。
- **対策（設計済み）**: 連携カードに状態表示+「再連携」1タップ導線。失効しても他機能は動くフェイルセーフにする。
