## 目的

ニュース・予定・メール・タスク・メモを1画面に集約する**個人用の秘書エージェント**Webアプリ。
旧「AIニュース自動配信システム（Slack配信）」を土台に作り替えた。**Slack配信は廃止**し、Webアプリに一本化。

## 認証（フェーズ1の中核）

- **全ページログイン必須**。`src/middleware.ts`（next-auth/middleware）が未ログインを `/login` へ送る
- 除外パス: `/api/auth`（NextAuth自身）、`/api/cron`（CRON_SECRETのBearer認証で別途保護）、`/login`、静的ファイル
- Googleサインイン後、`ALLOWED_EMAIL`（カンマ区切り可）に載っているメールだけ許可（`signIn` コールバック）
- フェイルセーフ: 認証系の環境変数が未設定なら誰もログインできず、ログイン画面がセットアップ案内を表示
- `(main)/layout.tsx` でもセッション確認（ミドルウェアとの二重防御）。Server Actionも各アクション先頭で確認
- ※ アプリログインと、フェーズ2のGoogle API連携（GoogleAccountテーブル）は**別物**として分離する設計

## API仕様

### GET/POST `/api/cron/fetch`（認証: Bearer CRON_SECRET）
毎時実行（GitHub Actions）。sources同期 → 各RSS取得 → 48時間以内の新着URLだけ `Article` に保存。
新規記事の英語タイトルはDeepLで日本語訳して `titleJa` に保存。
戻り: `{ ok, inserted, pending, details[] }`

### `/api/auth/[...nextauth]`
NextAuthの全エンドポイント（サインイン・コールバック・セッション）。設定は `src/lib/nextauth.ts`。

### GET `/api/google/connect?label=会社`（要ログイン）
Google連携の開始。CSRF対策のstateをhttpOnly Cookieに保存し、Googleの同意画面へリダイレクト。
スコープは `calendar.readonly` + `gmail.readonly`（読み取り専用のみ）。`access_type=offline&prompt=consent`。

### GET `/api/google/callback`（要ログイン）
コードをトークンに交換し、refresh_tokenを**AES-256-GCMで暗号化して** `GoogleAccount` にupsert
（同一メール=再連携で上書き）。完了後 `/settings?connected=` へ。失敗時は `/settings?error=` に戻す。

### Server Actions
- `src/app/actions.ts` … `translateArticle` / `summarizeArticle`（記事のオンデマンド翻訳・要約）
- `src/app/(main)/tasks/actions.ts` … `createTask` / `toggleTask` / `deleteTask`
- `src/app/(main)/memos/actions.ts` … `createMemo` / `updateMemo` / `promoteMemo` / `deleteMemo`
- `src/app/(main)/settings/google-actions.ts` … `disconnectGoogleAccount` / `saveCalendarSelection` / `renameGoogleAccount`
- すべて先頭でセッション確認（多層防御）

## 会議機能（録音・文字起こし・レポート）

- **録音**: ブラウザのMediaRecorder（Chrome=webm/opus 48kbps、Safari=mp4）。10秒ごとに分割回収し長時間でも安定
- **保存**: 音声はブラウザから **Vercel Blob へ直接アップロード**（サーバーレスの4.5MB制限回避。上限300MB≒約3時間）。
  URLはランダムサフィックス付き公開URL（推測不能だが公開である点に注意。将来privateへ移行検討）
- **文字起こし**: **Gemini API**（`gemini-2.5-flash`・思考オフ）。15MB以下はインライン、超えたらFiles API経由。
  話者ラベル付きで忠実に書き起こす。`GEMINI_API_KEY` 未設定でも録音・保存は動く（フェイルセーフ）
- **レポート要約**: **Claude Haiku**。概要表・要点・決定事項・TODO表・議論詳細のMarkdownを生成し、
  react-markdown（GFM）+ `.report` スタイルで整形表示
- **カレンダー自動登録**: 保存時に `calendar.events` 権限を持つアカウントのメインカレンダーへ
  「会議: タイトル」を録音開始時刻〜（長さ or 1時間）で登録。権限が無ければ会議だけ保存し、詳細画面から再試行可
- **処理ルート**: 文字起こし `/api/meetings/[id]/transcribe`（maxDuration 300）、
  レポート `/api/meetings/[id]/summarize`（120）。アップロード許可証は `/api/meetings/upload`（要ログイン）
- **削除**: 会議削除でBlobの音声も削除

## 朝のブリーフィング（フェーズ3）

- **生成**: `src/lib/briefing.ts`。今日の予定（全カレンダー）＋未完了タスク＋受信メール（全アカウント最新5件×N）＋
  最新ニュース8本を材料に、Claude Haiku が Markdown で生成（1200字以内・与えられた情報のみ・推測禁止）
- **保存**: `Briefing`（dateKey unique・1日1件、再生成は上書き）
- **実行経路**: 毎朝7時JST = GitHub Actions → `GET /api/cron/briefing`（CRON_SECRET Bearer）/
  ダッシュボードの「今すぐ生成/更新」ボタン → `POST /api/briefing`（要ログイン）
- **表示**: ダッシュボード①カードで react-markdown（`.report` スタイル）レンダリング

## ニュースの既読管理

- **方式**: 記事は削除せず `Article.readAt` に既読日時を入れて一覧から隠す（履歴が残り、いつでも見返せる）
- **未読が既定**: /news・/news/[ジャンル]・ダッシュボードのニュースカードはすべて未読のみ表示。ジャンル件数も未読ベース
- **操作**: 各記事の ✓ で既読（既読一覧では ↩ で未読に戻す）／「表示中の◯件を既読に」で一括処理
- **お気に入りは別軸**: ★は既読・未読を問わず `?filter=fav` に残る（読み終えても保存され続ける）

## ニュースの好み学習（おすすめ）

- **学習**: ニュース画面の「好みを学習」→ お気に入り記事（直近50件）のタイトル・本文抜粋を Claude Haiku に渡し、
  関心キーワード（10〜20語）と傾向の説明をJSONで抽出 → `NewsPreference`（id="default"の1レコード）に保存
- **おすすめ判定**: 保存キーワードと記事テキスト（和文タイトル＋原題＋本文抜粋）の一致数でスコアリングし、
  スコア降順→新着順に上位30件を表示（`?filter=reco`）。AI呼び出しは学習時のみでコストは限定的
- **前提**: お気に入りが3件以上（`MIN_FAVORITES_TO_LEARN`）。未学習なら「おすすめ」チップは出さない
- **収集の共通化**: `src/lib/news-fetch.ts` を cron（毎時）と手動取得ボタン（`POST /api/news/fetch`・要ログイン）で共用

## プッシュ通知（Web Push）

- **有効化**: 設定ページ「プッシュ通知」→ 端末ごとに許可（購読は `PushSubscription` に保存・テスト送信あり）
- **送信**: 毎朝のブリーフィング完成時（cron）に全端末へ通知（`src/lib/push.ts`）。無効化された購読（410）は自動削除
- **鍵**: VAPID（`NEXT_PUBLIC_VAPID_PUBLIC_KEY` / `VAPID_PRIVATE_KEY` / `VAPID_SUBJECT`）。未設定なら送らないだけ
- **サービスワーカー**: `public/sw.js`（push表示・タップでアプリを開く）
- **iOS**: 「ホーム画面に追加」したPWAから開いた場合のみ有効（iOS 16.4+の仕様）

## メモの拡張機能

- **録音→文字起こし**: メモ入力欄の録音ボタン。停止するとBlobへ保存→Gemini文字起こし→本文に自動挿入（音声も添付として残る）
- **添付**: 画像・動画・PDF・音声を1メモ10個まで（Private Blob・上限200MB/個）。画像サムネイル・動画/音声プレーヤー・PDFリンクで表示（署名付きURL・1時間）
- **URLリンク化**: 本文中の http(s) URLを自動でリンク表示（`src/components/Linkify.tsx`）
- **フォルダ**: quick/pinned共通のグルーピング（`MemoFolder`）。削除時は中のメモが未分類に戻る（SetNull）。メモ削除時は添付の実体もBlobから削除
- 関連ルート: `/api/uploads`（添付の許可証発行）・`/api/memos/transcribe`（録音の文字起こし）。どちらも要ログイン
- ダッシュボードの突発メモカードはテキスト即入力のまま（フル機能は/memos）

## Google連携（フェーズ2）

- **方式**: アプリログイン（NextAuth）とは別の自前OAuthフロー。1ユーザーがN個のGoogleアカウント（会社/個人）を
  ラベル付きで接続できる。実装: `src/lib/google/`（oauth.ts / api.ts / calendar.ts / gmail.ts）
- **トークン管理**: refresh_tokenのみDB保存（暗号化）。アクセストークンはメモリキャッシュ＋自動更新。
  `invalid_grant`（7日失効・取り消し）で `status="expired"` に自動マーク → 画面に「要再連携」表示＋1タップ再連携
- **今日の予定**: 各アカウントの選択カレンダー（`calendarIds`。既定はprimary）からJSTの当日分を取得。
  繰り返し予定は展開（singleEvents）、同一予定の重複はタイトル+開始時刻で排除、終日→時刻順に統合表示。
  家族共用カレンダーは「個人アカウントに共有されているカレンダー」を選択して表示する（別OAuth不要）
- **メール**: 受信トレイの最新メールのメタデータ（件名・差出人・抜粋・未読）だけ取得。
  **本文はDBにも画面にも保存しない**。開くときはGmail本体へリンク遷移
- **フェイルセーフ**: 1アカウント・1カレンダーの失敗は他を止めない。環境変数不足時は連携ボタンの代わりに案内を表示

## DBスキーマ（prisma/schema.prisma）

- **Source / Article** … 旧システムから継続（`Article.status` の sent/skipped 更新は廃止。一覧は最新順表示）
- **SendLog** … 旧Slack配信の記録。参照コードは無し（履歴保全のため残置）
- **GoogleAccount** … フェーズ2用。連携Googleアカウント（label/email/暗号化refresh_token/scopes/status/calendarIds/colorHex）
- **Task** … title/note/due/priority(0-2)/status(open・done)/completedAt
- **Memo** … kind("quick"=突発 / "pinned"=よく使う)/title/body/sortOrder
- **Briefing** … dateKey("YYYY-MM-DD" JST・unique)/content。フェーズ3のcronが1日1件生成

## 翻訳（DeepL）と要約（Claude Haiku）

旧システムから継続。どちらも未設定でも壊れないフェイルセーフ。

## セキュリティ

- 全ページログイン必須（ミドルウェア + レイアウト + Server Action の多層防御）
- Cronエンドポイント: `CRON_SECRET` Bearer認証・**未設定なら常に拒否**・定数時間比較
- 秘密情報はすべて環境変数。ログイン画面・設定画面は「未設定のキー名」だけ表示し、値は絶対に出さない
- RSS由来のURLは http/https のみ許可
- Next.jsは14.2.35（ミドルウェア認証バイパス CVE-2025-29927 等のパッチ適用済み）
- 個人用のため noindex

## 今後の拡張（フェーズ2〜3の設計要点）

- Google連携は自前OAuthフロー（`/api/google/connect` → callback）。refresh_tokenはAES-256-GCMで暗号化しGoogleAccountへ
- 家族共用カレンダーは個人アカウントのcalendarList経由で選択（別OAuth不要）
- GCP同意画面はテストモード運用 → **refresh tokenが7日で失効**するため「再連携」ボタンを設ける
- メール本文はDBに保存しない（都度取得+短期キャッシュ）
- ブリーフィング: 朝7時JSTにGitHub Actionsから `/api/cron/briefing` を叩き、Briefingへ保存
